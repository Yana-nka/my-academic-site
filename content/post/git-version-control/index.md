## Что такое Git?

Git — это распределённая система контроля версий, разработанная Линусом Торвальдсом в 2005 году. Она позволяет отслеживать изменения в файлах, синхронизировать работу нескольких участников проекта и откатывать изменения при необходимости.

## Основные понятия

- **Репозиторий** — хранилище файлов и истории изменений
- **Коммит (commit)** — сохранение изменений с комментарием
- **Ветка (branch)** — параллельная линия разработки
- **Слияние (merge)** — объединение изменений из разных веток
- **Удалённый репозиторий (remote)** — репозиторий на сервере (например, GitHub)

## Git в научной работе

Git полезен для:
1. Хранения версий статей и отчётов
2. Совместной работы над проектами
3. Публикации кода и данных
4. Создания персональных сайтов (GitHub Pages)

## Основные команды

```bash
git init                    # Создать новый репозиторий
git add .                   # Добавить файлы в индекс
git commit -m 'Сообщение'   # Создать коммит
git push                    # Отправить на удалённый репозиторий
git pull                    # Получить изменения с сервера
git branch                  # Показать ветки
git merge                   # Слить ветки
```

## Заключение

Git — необходимый инструмент для современного научного работника. Он позволяет организовать аккуратную работу с файлами, вести историю изменений и публиковать результаты.
